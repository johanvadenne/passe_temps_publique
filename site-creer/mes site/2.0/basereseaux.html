Réseaux

Un réseaux est composée de plusieurs appareils
qui sont connecté entre eux, pour les identifiés chaque appareils
détient une adresse IP.

IP

IP qui veut dire Internet Protocol, est un moyen de
reconnaitre l'appareil dans le réseaux afin de lui transmettre ou
envoyé les données qui demande. L'IP est constitué d'une suite de chiffre, il y a :

IPV4:
L'IPV4 est constitué de 4 octets, il se renouvelle souvent pour éviter
les problèmes (on appelle ça des adresse dynamique),
car il y a pas assée d'adresse IPV4 pour toutes les personnes dans le monde.
Exemple d'IPV4: 192.168.23.5

IPV6:
L'IPV6 est constitué de 16 octets, cette adresse est fixe elle ne change
pas comparé à l'IPV4, car elle est unique.
Exemple d'IPV6: 2001:0db8:0000:85a3:0000:0000:ac1f:8001

Pour connaitre son adresse IP, il faut lancer l'invite de commande et taper
ipconfig, et pour linux ifconfig.

Il y a deux types IP :

Les IP privées: ce sont des adresses fourni par un router, ce router permet de
faire le lien entre le réseaux privée et le réseau public.
Ces adresse sont utilisé dans des entreprises, écoles, maison...

Les IP publics: ce sont des adresses qui circule dans le monde entier, est sont
accessible à travers internet.

passerelle

une passerelle est le nom générique d'un dispositif permettant de relier deux réseaux
informatiques de types différents, par exemple un réseau local et le réseau Internet.

masque de sous réseaux

Un sous-réseau est une subdivision logique d'un réseau de taille plus importante.
Le masque de sous-réseau permet de distinguer la partie de l'adresse commune à tous les appareils
du sous-réseau et celle qui varie d'un appareil à l'autre. Un sous-réseau correspond typiquement à
un réseau local sous-jacent.

Exemple :

un masque de sous réseaux : /24 = 255.255.255.0

DHCP

Le protocole DHCP (Dynamic Host Configuration Protocol) est un
protocole client/serveur qui fournit automatiquement une adresse IP et d'autres
informations de configuration associées, telles que le masque de sous-réseau et la
passerelle par défaut.
Le DHCP aura une pool d'adresse définit, une pool d'adresse c'est un nombre d'IP qu'on
autorise au DHCP de fournir.

Exemple :

je lui définit une pool entre : 192.168.0.0 à 192.168.1.0
cela veut dire que mon DHCP peut fournir 257 adresse IP.
On peut attribué plusieurs pool d'adresse si on le veut.

DNS

Le Domain Name System ou DNS est un service informatique qui traduit
les noms de domaine Internet en adresse IP ou autres enregistrements.
Lorsque vous aller recherché google.fr sur internet, la machine ne comprend pas.
donc pour cela le DNS va associé une adresse IP à google.fr, et quand vous allez
chercher google.fr le DNS traduit afin de comprendre votre recherche et vous envoyer
le site que vous avez demandé.
